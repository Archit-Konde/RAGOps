How It Works

1
Ingest Upload a .txt, .md, or .pdf file via POST /v1/ingest. The pipeline chunks the text, generates embeddings with sentence-transformers, and stores everything in PostgreSQL + pgvector.
2
Query Ask a question via POST /v1/query. The pipeline embeds your query, performs vector similarity search, reranks results with a cross-encoder, and generates an answer using an LLM.
3
Get Answers Receive a grounded answer with source attribution — every claim is traced back to specific document chunks.
